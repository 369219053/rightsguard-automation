## 🔍 自动化原理

### Accessibility Service工作原理

Accessibility Service是Android系统提供的无障碍服务框架,主要用于帮助残障人士使用Android设备。但同时也可以用于UI自动化。

#### 核心机制
```
┌─────────────────────────────────────┐
│      Android系统                     │
│  ┌─────────────────────────────┐   │
│  │  Accessibility Manager      │   │
│  └─────────────────────────────┘   │
│           ↓ 事件分发                │
│  ┌─────────────────────────────┐   │
│  │  Accessibility Service      │   │
│  │  (自动化服务)                │   │
│  └─────────────────────────────┘   │
│           ↓ 获取UI信息              │
│  ┌─────────────────────────────┐   │
│  │  AccessibilityNodeInfo      │   │
│  │  (UI节点树)                  │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
         ↓ 控制
┌─────────────────────────────────────┐
│      目标应用 (权利卫士)             │
└─────────────────────────────────────┘
```

#### 关键能力
1. **监听UI事件** - 窗口变化、内容变化、焦点变化等
2. **获取UI信息** - 控件层级、属性、文本、位置等
3. **模拟用户操作** - 点击、滑动、输入、手势等
4. **跨应用操作** - 可以控制任何应用的UI

### 技术优势

| 特性 | Accessibility Service | UI Automator | Instrumentation |
|------|---------------------|--------------|-----------------|
| **无需Root** | ✅ | ✅ | ✅ |
| **跨应用** | ✅ | ✅ | ❌ |
| **后台运行** | ✅ | ❌ | ❌ |
| **实时监控** | ✅ | ❌ | ✅ |
| **用户授权** | 需要 | 不需要 | 不需要 |

```java
private void clearClipboard() {

        if (clipboard != null) {
```

#### 3. 状态检测与判断 (V1.4优化)

**问题**: 系统录屏权限弹窗有两种状态
- 状态1: "单个应用" + "继续"按钮
- 状态2: "整个屏幕" + "立即开始"按钮

**核心难点**:
1. 按钮文本无法通过`getText()`获取(可能显示在子节点)
2. 点击"单个应用"后如果不return,会立即执行后续代码导致误点击
3. 需要合理的等待时间让界面完全更新

**解决方案 (V1.4)**:
```java
// 1. 先查找"单个应用"文本
List<AccessibilityNodeInfo> spinnerNodes =
    rootNode.findAccessibilityNodeInfosByText("单个应用");

if (spinnerNodes != null && !spinnerNodes.isEmpty()) {
    // 找到"单个应用",点击下拉框
    boolean clicked = spinnerNodes.get(0).performAction(ACTION_CLICK);

    if (clicked) {
        logD("✅ 成功点击'单个应用'下拉框,等待下拉菜单展开...");

        // 在新线程中等待1500ms后点击"整个屏幕"
        new Thread(() -> {
            Thread.sleep(1500);
            clickWholeScreenOption();
        }).start();

        // 重要: 立即返回,不要继续执行后面的代码!
        rootNode.recycle();
        return;
    }
} else {
    // 未找到"单个应用",检查是否已经是"整个屏幕"模式
    List<AccessibilityNodeInfo> wholeScreenNodes =
        rootNode.findAccessibilityNodeInfosByText("整个屏幕");

    if (wholeScreenNodes != null && !wholeScreenNodes.isEmpty()) {
        logD("✅ 已经是'整个屏幕'模式,查找确认按钮");
        // 通过ID查找并点击button1
        findAndClickButtonById(rootNode, "button1");
    }
}
```

### 详细步骤说明

#### 步骤1: 启动自动化服务

**目的**: 初始化Accessibility Service

**配置文件** (`res/xml/accessibility_service_config.xml`):
```xml
<?xml version="1.0" encoding="utf-8"?>
<accessibility-service xmlns:android="http://schemas.android.com/apk/res/android"
    android:accessibilityEventTypes="typeAllMask"
    android:accessibilityFeedbackType="feedbackGeneric"
    android:accessibilityFlags="flagDefault|flagReportViewIds|flagRetrieveInteractiveWindows"
    android:canPerformGestures="true"
    android:canRetrieveWindowContent="true"
    android:description="@string/accessibility_service_description"
    android:notificationTimeout="100" />
```

**⚠️ 重要配置说明**:

1. **不要限制packageNames**:
   - ❌ 错误: `android:packageNames="com.android.systemui"`
   - ✅ 正确: 不设置`packageNames`属性,监听所有应用

2. **必须包含的flags**:
   - `flagReportViewIds` - 允许获取View的Resource ID
   - `flagRetrieveInteractiveWindows` - 允许获取窗口内容

3. **必须的权限**:
   - `canPerformGestures="true"` - 允许执行手势操作
   - `canRetrieveWindowContent="true"` - 允许读取窗口内容

**代码示例**:
```java
public class AutomationAccessibilityService extends AccessibilityService {

    @Override
    public void onCreate() {
        super.onCreate();
        instance = this;
        Log.d(TAG, "无障碍服务已创建");
    }

    @Override
    public void onAccessibilityEvent(AccessibilityEvent event) {
        if (!isRunning) {
            return;
        }

        String packageName = event.getPackageName() != null ?
            event.getPackageName().toString() : "";

        // 只处理权利卫士应用的事件
        if (!TARGET_PACKAGE.equals(packageName)) {
            return;
        }

        int eventType = event.getEventType();
        Log.d(TAG, "收到事件: " + AccessibilityEvent.eventTypeToString(eventType));

        // 处理不同的事件类型
        if (eventType == AccessibilityEvent.TYPE_WINDOW_STATE_CHANGED) {
            handleWindowStateChanged(event);
        }
    }
}
```

**完整流程**:

**代码示例**:

```java
public void startAutomation() {
    Log.d(TAG, "启动自动化");
    isRunning = true;

    // 最小化当前应用(返回桌面)
    minimizeCurrentApp();

    // 延迟打开应用

/**
 * 最小化当前应用(返回桌面)
 */
private void minimizeCurrentApp() {
    try {
        Log.d(TAG, "最小化当前应用");
        Intent homeIntent = new Intent(Intent.ACTION_MAIN);
        homeIntent.addCategory(Intent.CATEGORY_HOME);
        homeIntent.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
        startActivity(homeIntent);
        Log.d(TAG, "已返回桌面");
    } catch (Exception e) {
        Log.e(TAG, "最小化失败: " + e.getMessage(), e);
    }
}

/**
 * 延迟打开应用
 */
private void delayedOpenApp() {
    DelayThread thread = new DelayThread();
    thread.start();
}

/**
 * 延迟线程
 */
private static class DelayThread extends Thread {
    @Override
    public void run() {
        try {
            Thread.sleep(500);
            if (instance != null) {
                instance.openTargetApp();
            }
        } catch (Exception e) {
            Log.e(TAG, "延迟打开应用失败: " + e.getMessage());
        }
    }
}

/**
 * 打开目标应用
 */
private void openTargetApp() {
    try {
        Log.d(TAG, "尝试打开权利卫士应用: " + TARGET_PACKAGE);

        // 使用显式Intent启动
        Intent intent = new Intent();
        intent.setClassName(TARGET_PACKAGE, TARGET_ACTIVITY);
        intent.setAction(Intent.ACTION_MAIN);
        intent.addCategory(Intent.CATEGORY_LAUNCHER);
        intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
        intent.addFlags(Intent.FLAG_ACTIVITY_CLEAR_TOP);

        startActivity(intent);
        Log.d(TAG, "成功启动权利卫士应用");

    } catch (Exception e) {
        Log.e(TAG, "启动权利卫士失败: " + e.getMessage(), e);
    }
}
```

**关键技术点**:

1. **最小化应用**: 使用`Intent.ACTION_MAIN`和`Intent.CATEGORY_HOME`返回桌面
2. **延迟启动**: 使用静态内部类`DelayThread`避免匿名内部类编译错误
3. **显式Intent**: 使用`setClassName()`明确指定Activity,比`getLaunchIntentForPackage()`更可靠
4. **Activity名称**: 权利卫士的启动Activity是`cn.tsa.activity.SplashActivity`

**常见问题**:

- ❌ `getLaunchIntentForPackage()`可能返回null

### OCR识别技术 (V3.0+)

**使用场景**: 当Accessibility Service无法dump到页面UI信息时(如抖音"更多"弹窗)

**技术方案**: Google ML Kit中文OCR识别

```
   ↓

   ↓

   ↓

   ↓

   ↓
6. 执行点击操作
```

#### 技术实现

```java
// 1. 初始化中文OCR识别器
import com.google.mlkit.vision.text.chinese.ChineseTextRecognizerOptions;

TextRecognizer recognizer = TextRecognition.getClient(
    new ChineseTextRecognizerOptions.Builder().build()
);

recognizer.process(image)
    .addOnSuccessListener(visionText -> {
        // 遍历所有文本块
        for (Text.TextBlock block : visionText.getTextBlocks()) {
            if (block.getText().contains("资质规则")) {
                // 找到目标文字,获取坐标
                Rect bounds = block.getBoundingBox();
                int x = bounds.centerX();
                int y = bounds.centerY();

            }
        }
    });
```

### 智能等待机制

**问题**: UI加载需要时间,直接查找可能失败

```java
private AccessibilityNodeInfo waitForElement(String resourceId, int maxWaitSeconds) {
    int waitCount = 0;
    int maxWaitCount = maxWaitSeconds * 10; // 每100ms检查一次

    while (waitCount < maxWaitCount) {
        AccessibilityNodeInfo rootNode = getRootInActiveWindow();
        if (rootNode != null) {
            List<AccessibilityNodeInfo> nodes =
                rootNode.findAccessibilityNodeInfosByViewId(resourceId);

            if (nodes != null && !nodes.isEmpty()) {
                Log.d(TAG, "找到元素: " + resourceId);
                return nodes.get(0);
            }
        }

        SystemClock.sleep(100);
        waitCount++;
    }

    Log.e(TAG, "等待元素超时: " + resourceId);
    return null;
}
```

```java
        rootNode.findAccessibilityNodeInfosByViewId(CONTINUE_BUTTON_ID);

    if (buttonNodes != null && !buttonNodes.isEmpty()) {
        CharSequence buttonText = buttonNodes.get(0).getText();

        // 如果是"立即开始",直接点击
        if (buttonText != null && buttonText.toString().contains("立即开始")) {
            buttonNodes.get(0).performAction(ACTION_CLICK);
            Log.d(TAG, "成功点击'立即开始'按钮");
            return;
        }
    }

    // 检测"单个应用"下拉框
    List<AccessibilityNodeInfo> spinnerNodes =
        rootNode.findAccessibilityNodeInfosByViewId(SCREEN_SHARE_MODE_SPINNER_ID);

    if (spinnerNodes != null && !spinnerNodes.isEmpty()) {
        CharSequence text = spinnerNodes.get(0).getText();

        // 如果是"单个应用",点击打开菜单
        if (text != null && text.toString().contains("单个应用")) {
            spinnerNodes.get(0).performAction(ACTION_CLICK);
            Log.d(TAG, "成功点击'单个应用'下拉框");

            // 异步点击"整个屏幕"
            new Thread(() -> {
                try {
                    Thread.sleep(500);
                    clickWholeScreenOption();
                } catch (Exception e) {
                    Log.e(TAG, "等待下拉菜单异常: " + e.getMessage());
                }
            }).start();
        }
    }
}
```

**3. 智能父节点查找**

```java
private void clickWholeScreenOption() {
    AccessibilityNodeInfo rootNode = getRootInActiveWindow();

    // 查找"整个屏幕"文本节点
    List<AccessibilityNodeInfo> nodes =
        rootNode.findAccessibilityNodeInfosByText("整个屏幕");

    if (nodes != null && !nodes.isEmpty()) {
        AccessibilityNodeInfo node = nodes.get(0);

        // 如果节点本身可点击,直接点击
        if (node.isClickable()) {
            node.performAction(ACTION_CLICK);
            Log.d(TAG, "成功点击'整个屏幕'节点");
            return;
        }

        // 如果节点不可点击,尝试点击父节点
        AccessibilityNodeInfo parent = node.getParent();
        if (parent != null && parent.isClickable()) {
            parent.performAction(ACTION_CLICK);
            Log.d(TAG, "成功点击'整个屏幕'父节点");
            return;
        }

        // 如果父节点也不可点击,尝试祖父节点
        AccessibilityNodeInfo grandParent = parent.getParent();
        if (grandParent != null && grandParent.isClickable()) {
            grandParent.performAction(ACTION_CLICK);
            Log.d(TAG, "成功点击'整个屏幕'祖父节点");
        }
    }
}
```

```
  └─ LinearLayout (menu_content)
```

### 状态管理代码

```java
public enum RecordingState {
    IDLE,           // 空闲
    LAUNCHING,      // 启动中
    READY,          // 就绪
    STARTING,       // 启动录屏中
    RECORDING,      // 录屏中
    STOPPING,       // 停止中
    COMPLETED,      // 完成
    ERROR           // 错误
}

public class StateManager {
    private RecordingState currentState = RecordingState.IDLE;
    private List<StateChangeListener> listeners = new ArrayList<>();

    public void setState(RecordingState newState) {
        RecordingState oldState = currentState;
        currentState = newState;

        Log.d(TAG, "状态变化: " + oldState + " -> " + newState);

        // 通知监听器
        for (StateChangeListener listener : listeners) {
            listener.onStateChanged(oldState, newState);
        }
    }

    public RecordingState getState() {
        return currentState;
    }
}
```

## ⚠️ 错误处理

### 常见错误场景

#### 0. 无法接收事件(最常见)

**现象**:
- 日志中没有"收到事件"的记录
- 自动化功能完全不工作
- 应用启动后没有任何自动操作

**原因**:
无障碍服务配置文件中限制了`packageNames`,导致无法接收目标应用的事件

**错误配置**:
```xml
<!-- ❌ 错误: 只监听系统UI -->
<accessibility-service
    android:packageNames="com.android.systemui" />
```

**正确配置**:
```xml
<!-- ✅ 正确: 不限制packageNames,监听所有应用 -->
<accessibility-service
    android:accessibilityEventTypes="typeAllMask"
    android:accessibilityFeedbackType="feedbackGeneric"
    android:accessibilityFlags="flagDefault|flagReportViewIds|flagRetrieveInteractiveWindows"
    android:canPerformGestures="true"
    android:canRetrieveWindowContent="true"
    android:description="@string/accessibility_service_description"
    android:notificationTimeout="100" />
    <!-- 注意: 没有 android:packageNames 属性 -->
```

**解决步骤**:
1. 修改配置文件,移除`packageNames`限制
2. 重新编译安装APK
3. **重新开启无障碍服务** (关闭再打开)
4. 测试是否能接收到事件

**验证方法**:
```bash
# 查看日志,应该能看到"收到事件"
adb logcat | grep "收到事件"
```

```java

    dumpButton.setOnClickListener(v -> dumpCurrentUI());

// 2. 获取当前窗口的UI结构
private void dumpCurrentUI() {
    AccessibilityNodeInfo rootNode = getRootInActiveWindow();
    if (rootNode == null) {
        showToast("无法获取UI结构");
        return;
    }

    // 3. 递归遍历UI树
    StringBuilder sb = new StringBuilder();
    sb.append("=== UI结构 Dump ===\n");
    sb.append("包名: ").append(rootNode.getPackageName()).append("\n");
    sb.append("时间: ").append(getCurrentTime()).append("\n\n");

    dumpNode(rootNode, sb, 0);

    // 4. 显示dump结果
    showDumpResult(sb.toString());
}

// 5. 递归遍历节点
private void dumpNode(AccessibilityNodeInfo node, StringBuilder sb, int depth) {
    if (node == null) return;

    // 缩进
    for (int i = 0; i < depth; i++) {
        sb.append(i == depth - 1 ? "├─ " : "│   ");
    }

    // 节点信息
    sb.append("[").append(node.getClassName()).append("]");
    sb.append(" (clickable=").append(node.isClickable()).append(")\n");

    // Resource ID
    String viewId = node.getViewIdResourceName();
    if (viewId != null && !viewId.isEmpty()) {
        for (int i = 0; i < depth; i++) sb.append("│   ");
        sb.append("ID: ").append(viewId).append("\n");
    }

    // 文本内容
    CharSequence text = node.getText();
    if (text != null && text.length() > 0) {
        for (int i = 0; i < depth; i++) sb.append("│   ");
        sb.append("Text: \"").append(text).append("\"\n");
    }

    // 内容描述
    CharSequence desc = node.getContentDescription();
    if (desc != null && desc.length() > 0) {
        for (int i = 0; i < depth; i++) sb.append("│   ");
        sb.append("Desc: \"").append(desc).append("\"\n");
    }

    // 位置和大小
    Rect bounds = new Rect();
    node.getBoundsInScreen(bounds);
    for (int i = 0; i < depth; i++) sb.append("│   ");
    sb.append("Bounds: ").append(bounds.toShortString()).append("\n");

    // 遍历子节点
    int childCount = node.getChildCount();
    for (int i = 0; i < childCount; i++) {
        AccessibilityNodeInfo child = node.getChild(i);
        dumpNode(child, sb, depth + 1);
        if (child != null) child.recycle();
    }
}

// 6. 显示dump结果
private void showDumpResult(String dumpText) {
    Intent intent = new Intent(this, DumpResultActivity.class);
    intent.putExtra("dump_text", dumpText);
    intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
    startActivity(intent);
}
```

## 🎯 坐标测试工具 (V2.8新增)

### 功能介绍

坐标测试工具是一个**交互式的坐标获取工具**,用于精确测试屏幕上任意按钮的坐标位置。

### 使用场景

当自动化流程中需要点击某个按钮,但无法通过无障碍服务找到该按钮时(例如Canvas绘制的按钮),可以使用坐标测试工具获取精确坐标。

### 使用方法

#### 1. 启动测试模式

- 打开悬浮窗
- 点击 **"📍测试"** 按钮
- 屏幕变成半透明,显示"坐标测试模式"提示

#### 2. 获取坐标

- 打开目标应用(如抖音订单详情页)
- 点击需要测试的按钮位置
- 屏幕中央弹出卡片,显示X、Y坐标

#### 3. 保存坐标

- 点击 **"使用"** 按钮保存坐标
- 或点击 **"取消"** 重新测试
- 右下角的 **"✕"** 按钮可以关闭测试模式

#### 4. 自动使用

- 保存后,自动化流程会优先使用测试的坐标
- 如果没有保存坐标,会使用默认坐标
